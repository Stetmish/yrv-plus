# yrv-plus: RISC-V ядро yrv с поддержкой плат DE10-Lite и T-Core

Этот репозиторий является форком проекта [yrv-plus](https://github.com/yuri-panchul/yrv-plus) от Юрия Панчула.

Исходное RISC-V ядро yrv разработано Monte J. Dalrymple и подробно описано в книге **"Inside An Open-Source Processor"** (ISBN 978-3-89576-443-1)[reference:0]. Ядро реализует набор инструкций **RV32IAC** (базовый набор целочисленных инструкций, атомарные операции и сжатые инструкции), а также большую часть расширения **B** (битовые операции)[reference:1].

## Структура проекта

Проект организован следующим образом:

*   **[`Plus/design/`](Plus/design)** — исходные коды RISC-V ядра `yrv_mcu` и всей его внутренней логики. Этот код **не изменялся** в рамках данной работы.
*   **[`Plus/boards/`](Plus/boards)** — содержит файлы для конкретных плат[reference:2]:
    *   [`de10_lite/`](Plus/boards/de10_lite) — порт для платы Terasic DE10-Lite[reference:3].
    *   [`t_core/`](Plus/boards/t_core) — порт для платы Terasic T-Core[reference:4].
*   **[`Plus/scripts/`](Plus/scripts)** — универсальные bash-скрипты для автоматизации сборки, программирования и загрузки программ[reference:5].
*   **[`Plus/programs/`](Plus/programs)** — примеры программ для тестирования ядра[reference:6].
*   **[`Plus/testbench/`](Plus/testbench)** — тестовое окружение для симуляции ядра[reference:7].

### 1. Плата DE10-Lite

Папка [`Plus/boards/de10_lite/`](Plus/boards/de10_lite) содержит полный набор файлов для запуска ядра на данной плате[reference:8]:
*   **`top.sv`**: Верхний модуль, связывающий ядро с периферией платы.[reference:9]
*   **`top.qsf`**: Файл настроек Quartus Prime с назначением выводов и параметрами проекта[reference:10].
*   **`top.sdc`**: Файл временных ограничений[reference:11].
*   **Скрипты**: `01_clean.bash`, `05_synthesize_for_fpga.bash`, `06_configure_fpga.bash`, `07_upload_soft_to_fpga.bash` для автоматизации[reference:12].

### 2. Плата T-Core

Папка [`Plus/boards/t_core/`](Plus/boards/t_core) содержит адаптированную версию для платы T-Core[reference:13]:
*   **`top.sv`**: Упрощённый верхний модуль. Удалены несуществующие интерфейсы. Добавлены поддержка 4-х пользовательских светодиодов, управление 4-мя встроенными RGB-светодиодами WS2812B через аппаратный драйвер, работа с 8-битным портом TMD GPIO.[reference:14]
*   **`top.qsf`**: Файл настроек Quartus с обновлённым назначением выводов в соответствии с документацией T-Core[reference:15].
*   **`top.sdc`**: Адаптированный файл временных ограничений[reference:16].
*   **Скрипты**: Скопированы из папки DE10-Lite без изменений, так как вся платформенно-зависимая информация вынесена в файлы проекта[reference:17].

## Как использовать

### Предварительные требования

1.  Установленный **Quartus Prime Standard Edition** (рекомендуется версия 18.1 или новее) с поддержкой семейства **MAX 10**.
2.  Установленный **RISC-V тулчейн** (например, из репозитория [RISC-V GNU Toolchain](https://github.com/riscv-collab/riscv-gnu-toolchain)) или использование встроенного в проект **RARS** (RISC-V Assembler and Runtime Simulator).
3.  Подключенный программатор **USB-Blaster II** и установленные драйверы.
# Инструкция по синтезу и загрузке проекта на FPGA

## 1. Переход в папку нужной платы

Выберите команду в зависимости от вашей платы:

- Для **DE10‑Lite**:
  ```bash
  cd Plus/boards/de10_lite

- Для **T‑Core**:
  ```bash
  cd Plus/boards/t_core
  
## 2. Синтез проекта и создание битстрима (`.sof`)

Запустите скрипт синтеза:

```bash
./05_synthesize_for_fpga.bash
```
Этот скрипт запускает полный цикл синтеза в Quartus.
В случае успеха он автоматически перейдёт к следующему шагу.

## 3. Программирование FPGA

Загрузите полученный битстрим в FPGA:

```bash
./06_configure_fpga.bash
```
Скрипт найдёт подключённый USB‑Blaster и загрузит файл top.sof в FPGA.

## 4. Загрузка пользовательской программы

### Подготовка
- Скомпилируйте вашу программу для RISC‑V в файл `program.mem32`.

### Подключение USB‑UART адаптера
- Для **T‑Core** подключите адаптер к пинам:
  - `TMD_D[6]` — RX
  - `TMD_D[7]` — GND  
  (для DE10‑Lite используйте соответствующие пины, указанные в документации).

### Загрузка
Запустите скрипт загрузки:

```bash
./07_upload_soft_to_fpga.bash
```
Скрипт отправит файл program.mem32 через UART на скорости 115200 бод.
Встроенный загрузчик запишет программу в память ядра.

**Примечание:** Убедитесь, что все скрипты имеют права на выполнение (`chmod +x`), а Quartus и USB‑Blaster правильно установлены и распознаны системой.
